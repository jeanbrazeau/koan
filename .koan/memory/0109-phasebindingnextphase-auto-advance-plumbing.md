On 2026-04-26, Leon shifted koan's workflow control flow (`koan/lib/workflows.py`, `koan/phases/format_step.py`, `koan/web/mcp_endpoint.py`) from phase-end-trampoline yields to per-binding auto-advance. The change: `PhaseBinding` (frozen dataclass) gained an optional `next_phase: str | None = None` field; bindings populated across both `PLAN_WORKFLOW` and `MILESTONES_WORKFLOW` (intake -> plan-spec / milestone-spec, plan-spec -> plan-review, execute -> exec-review for both; review phases plan-review / milestone-review / exec-review and curation get None). `PhaseContext` gained `next_phase: str | None` and `suggested_phases: list[str]` fields populated at the `_step_phase_handshake` site from `workflow.get_binding(current_phase).next_phase` and `get_suggested_phases(workflow, current_phase)`. A pure helper `terminal_invoke(next_phase, suggested_phases) -> str` in `koan/phases/format_step.py` (no app_state or workflow imports; no dependency cycles) renders the last-step `invoke_after` directive: `koan_set_phase("X")` (auto-advance) when bound, `koan_yield(suggestions=[...])` (full yield) when None. The text includes an explicit override clause: "If exceptional circumstances warrant user direction, call koan_yield instead." The `format_phase_complete` trampoline in `koan/phases/format_step.py` was deleted; its call site in `koan/web/mcp_endpoint.py` (the `next_step is None` branch in `koan_complete_step`) was replaced with a defensive fallback message nudging the LLM back to the right path if it accidentally calls `koan_complete_step` at the boundary. Every phase module's last-step `step_guidance` now returns `StepGuidance(..., invoke_after=terminal_invoke(ctx.next_phase, ctx.suggested_phases))`. Auto-advance is GUIDANCE not enforcement -- the orchestrator may yield instead when warranted; documented in `docs/guided-transitions.md`. The yolo / `directed_phases` short-circuit is independent and continues to work in eval mode. The eval-fixture pinned snapshot at `evals/fixtures/koan-1/repo/` retains the old symbol intentionally because fixtures are pinned to a specific SHA.
