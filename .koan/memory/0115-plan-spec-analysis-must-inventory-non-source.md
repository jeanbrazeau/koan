This entry records a defect in a plan-spec analysis for an inline-review backend removal in koan. On 2026-04-26, the plan-spec deliberately scoped the change to delete `phase_summaries` from `koan/projections.py:Run`, the corresponding fold case, the `phase_summary_captured` event, and the capture block in `koan_yield`. The plan-spec's "Files NOT modified" list correctly named the eval-fixture pinned snapshot (`evals/fixtures/koan-1/repo/`, out of scope because fixtures are pinned to a specific SHA) and the compiled frontend bundles (`koan/web/static/app/assets/`, regenerated by build) but did NOT enumerate the active eval-harness modules (`evals/harvest.py`, `evals/scorers.py`, `evals/rubrics.py`, `evals/runner.py`, `tests/evals/conftest.py`, `tests/evals/test_koan.py`) which import koan types and consume the `phase_summaries` field directly via `store.projection.run.phase_summaries`.

The defect surfaced post-implementation only because Leon explicitly asked during curation: "investigate whether or not we accounted for the eval framework's use of summaries when we removed them" -- without that steering, the dogfooded run would have shipped with broken eval harvest (`AttributeError` on the deleted field at `evals/harvest.py:29`) and degraded scoring (placeholder text "(no summary captured for this phase)" returned for all per-phase summary-based scoring at `evals/scorers.py:243, 291`).

Root cause: the plan-spec analysis enumerated source-tree consumers (phase prompts, system prompts, permission fence) but did not separately enumerate non-source consumers (eval harness, scripts, integration tests, doc snippets). A follow-up change was scheduled to remove summary-based scoring from the eval framework entirely.

Lesson for future field-removal work in koan: when removing a field from `koan/projections.py`, `koan/state.py`, `koan/events.py`, or any koan-core type, plan-spec must inventory non-source consumers in addition to source-tree consumers. The audit checklist:

- `evals/` -- eval harness modules import koan types and read projection state directly.
- `scripts/` -- maintenance scripts that may parse koan state files.
- integration tests under `tests/evals/`, `tests/integration/`, etc.
- doc snippets in `docs/` that exemplify the field's shape.
- any `*.py` that imports from `koan.projections`, `koan.state`, or `koan.events`.

The "Files NOT modified" list in plan-spec must be exhaustive enough to cover all dependents.
